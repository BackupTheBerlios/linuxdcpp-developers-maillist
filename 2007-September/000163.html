It seems scons has some serious issues on *BSD. I've gotten a lot of complaints 
about it recently. Scons does all the dependency checks, so if it can't find 
your library and you do have it and the headers, then it is an issue with scons.

The patch you mention is very old and was applied at the date stated in the 
changelog. The FreeBSD package is kept up-to-date and reflects the latest cvs.
