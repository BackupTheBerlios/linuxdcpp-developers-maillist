I am not on the list, but I do read in the archive once a while.

You need to add /usr/local/include in CPPPATH and /usr/local/lib in  
LIBPATH to run scons. It's how linuxdcpp port in FreeBSD can build it. I  
haven't been able to update to the latest version of CVS, because  
linuxdcpp doesn't work very well with new glib/gtk as I am testing  
bleeding edge version on my desktop. I have glib 2.14.0, gtk 2.11.6 and  
gnome 2.19.6. The gtk 2.12.0 has been released recently and a lot of GNOME  
2.20 tarballs are out right now. I will see if linuxdcpp will working  
again with new glib/gtk.

If you still have stable version of glib/gtk, feel free to test  
http://people.freebsd.org/~mezz/diff/linuxdcpp.diff that will update to  
0.0.1.20070804. If it works for you, let me know and I shall commit it.

As for your comment of 'it's really buggy'. I won't be surpised if you  
will get same result if you install it on your own. LinuxDC++ is stable to  
me until with that new glib/gtk. However, if you want to kill that 'it's  
really buggy', I suggest you to start to make bug report with better  
details.
