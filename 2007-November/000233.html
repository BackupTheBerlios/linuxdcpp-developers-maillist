Hello,

I'm sorry, but I cannot include your patch in LinuxDC++ for a number of reasons. 
First, you modify the DC++ core (client/). We copy this directory from Windows 
DC++ so any changes to this have to generally be sent upstream first and 
approved there. Secondly, even if you sent this upstream, I know they will 
reject it. Namely because they specifically removed this functionality not too 
long ago. Their intent is to force users to upgrade to newer clients and 
technologies. We intend for LinuxDC++ to be a port DC++ to *nix, thus we try to 
keep our core in sync with theirs as much as we can.
