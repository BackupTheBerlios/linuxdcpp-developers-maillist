Hello all,

I switched the format of the changelog so that individual changes could be 
associated more easily with their release version. Also, I will be changing how 
I go about doing commits from now on. Instead of doing a large number of changes 
on my local copy then committing them all at once, I will be committing a fix as 
soon I've compiled and tested it. Thus each future CVS commit should only have 
one fix in it. This should allow users to get the very latest fixes and it 
should allow patch writers to create patches on top of the very latest code; 
making my life more easier when I merge their changes ;). If these CVS commit 
emails start spamming the list too often, I will disable them.

Regards,
Steven Sheehy
