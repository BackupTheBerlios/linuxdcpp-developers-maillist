This is an item in magnet's context menu. It opens new search tab and
starts search of magnet.

> Passing the magnet link as a command-line
> argument can probably be easily done in
> an OS-independant way. But I guess also some
> GUI additions are needed. Like the explicit
> "Open magnet link" dialog somewhere and maybe
> a new search option. But that's what I think
> was in an original patch, so I am not sure if

No, there was no 'open magnet link' in original patch. 

I thought about command line option, but faced with internal problem 
in DC core. If I remember right, there was no way to add entry in 
download queue if we don't know at least one source CID. So, the 
only option when we have magnet in hand is to open search tab and 
hope that we find some sources which can be added by user via 
right-click menu.

Better approach of handling magnet link passed via command line would 
be to add it to download queue without sources and let background
search find sources to download from. But this cannot be done without
internal DC core modification (at least I didn't find how to do this).
