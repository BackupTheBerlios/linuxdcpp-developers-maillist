> magnet:?xt=urn:sha1:YNCKHTQCWBTRNJIV4WNAE52SJUQCZO5C

AFAIK, the only hash dcpp core understands is Tiger. And the support of this one
has been mplemented, of course.

> What I would like to implement, is a
> text dialog for conviniently entering the
> magnet links, and the command-line argument

Such a command line option will be a plus. As for the dialog, I do not think
that we need two of them. I see two options:

1. Enhance the magnet properties dialog to make the url field of it
editable, make the dialog accessible from the menu, add a search button to
the dialog.

2. Add an entry field for the magnets to the search tab.
